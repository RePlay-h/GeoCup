
# ВЫШЕ КРЫШИ: ПОСТРОЕНИЕ ЕДИНОЙ МОДЕЛИ ВЫСОТНОСТИ ЗДАНИЙ

Решение для объединения разнородных геоданных зданий и построения единой, достоверной и интерпретируемой модели высотности с оценкой качества данных и дополнительными аналитическими признаками.

Проект разработан в рамках кейс-чемпионата Changellenge >> Cup IT 2026 (МТС).

---

## Описание проекта

В рамках задачи необходимо объединить два независимых источника геоданных по зданиям, устранить несоответствия, восстановить отсутствующие значения высоты и получить единый слой данных, пригодный для дальнейшего анализа и использования в прикладных задачах.

Основная сложность заключается в том, что:
- источники имеют разную структуру и полноту;
- отсутствует единый идентификатор зданий;
- высота присутствует только в одном источнике;
- данные содержат ошибки, выбросы и пространственные аномалии.

Решение направлено на создание надежного пайплайна обработки данных, который учитывает как физические, так и пространственные свойства объектов.

---

## Ключевая идея решения

В отличие от классических подходов, где недостающие значения восстанавливаются моделью, в данном проекте используется следующая логика:

- максимально используются реальные данные;
- выполняется многоуровневая очистка;
- применяется пространственный анализ;
- машинное обучение используется только как fallback.

Таким образом, обеспечивается баланс между точностью, интерпретируемостью и надежностью.

---

## Архитектура пайплайна

1. Источник A + Источник B

2. Унификация структуры данных

3. Очистка геометрии и атрибутов

4. Пространственное сопоставление (IoU + граф)
  
5. Формирование компонент зданий

6. Определение высоты (B → A → ML)

7. Формирование аналитических признаков

8. Экспорт итогового слоя

---

## Основные этапы обработки

### 1. Загрузка и унификация данных
- чтение CSV-файлов;
- приведение схем к единому виду;

---

### 2. Очистка данных

Реализована многоуровневая очистка:

**Физическая проверка**
- удаление невозможных значений высоты;
- проверка диапазонов этажности.

**Статистическая очистка (IQR)**
- выявление глобальных выбросов.

**Пространственная очистка (LISA)**
- выявление локальных аномалий относительно соседних зданий.

Важно: данные не удаляются, а помечаются флагами.

---

### 3. Пространственное сопоставление

- построение пространственного индекса (R-tree);
- поиск кандидатов;
- расчет IoU (Intersection over Union);
- построение графа;
- выделение компонент связности.

Поддерживаются сложные случаи:
- 1:1
- 1:N
- N:N

---

### 4. Определение высоты

Используется иерархический алгоритм:

1. Источник B (прямая высота)
2. Источник A (этажность → высота)
3. ML 

Дополнительно:
- проверяется согласованность данных;
- используется взвешенное среднее по площади;
- фиксируются конфликты между источниками.

---

### 5. Оценка качества данных

Для каждого здания рассчитывается:

**confidence_score**
- отражает надежность высоты;
- учитывает источник, согласованность и наличие конфликтов.

---

### 6. Дополнительные аналитические признаки

**dense_area**
- индикатор плотной застройки.

**eco_risk_score**
- интегральная оценка городской нагрузки;
- учитывает плотность, высоту, площадь и сложность структуры.

---

### 7. Машинное обучение

Используется только для объектов без высоты.

Модель:
- CatBoost (основная)
- LightGBM (сравнение)

Особенности:
- spatial cross-validation (GroupKFold);
- учет пространственной структуры;
- KNN-признаки (локальный контекст);
- геометрические признаки (площадь, форма, bbox).

---

## Выходные данные

Итоговый датасет содержит:

- height_final_full — итоговая высота
- height_source — источник значения
- confidence_score — достоверность
- eco_risk_score — экологическая нагрузка
- dense_area_score — плотная застройка
- geometry — геометрия здания

## Запуск
### 1.  Скачать Docker Desktop. Важно: после установки запустите его
### 2. Клонируем этот проект: 
```
git clone https://github.com/RePlay-h/GeoCup.git
cd GeoCup
```
### 3. 
```
mkdir data/raw
```
### 4. Поместите файлы cup_it_example_src_A.csv и cup_it_example_src_B.csv в папку data/raw
### 5. Генерация данных (Data Pipeline)
Запустите контейнер для обработки сырых геопространственных данных. Этот шаг подготавливает данные для дальнейшей визуализации.

```
docker compose run --rm data-pipeline
```

### 6. Сборка векторных тайлов (PMTiles)

После генерации данных необходимо создать слой векторных тайлов в формате PMTiles, который будет использоваться для отображения на карте.

```
docker compose run --rm tile-generator
```

### 7. Запуск веб-интерфейса

Запустите все сервисы проекта, включая веб-интерфейс, в фоновом режиме.

```
docker compose up -d frontend
```

### 8. Доступ к приложению

Откройте ваш веб-браузер и перейдите по следующему адресу для доступа к приложению:

```
http://localhost:5173
```

##  Полный цикл пересборки (Рекомендуется)

Если вы внесли изменения в логику обработки данных, код генератора тайлов или хотите выполнить гарантированно чистый запуск, рекомендуется выполнить полный цикл пересборки:

```bash
docker compose down
docker compose build tile-generator
docker compose run --rm data-pipeline
docker compose run --rm tile-generator
docker compose up -d
```

##  Справочник команд

| Команда                                   | Описание                                                              |
| :---------------------------------------- | :-------------------------------------------------------------------- |
| `docker compose down`                     | Остановить и удалить все активные контейнеры                          |
| `docker compose build tile-generator`     | Пересобрать генератор тайлов (необходимо после изменений в `tile.py`) |
| `docker compose logs -f frontend`         | Интерактивный просмотр логов фронтенда                                |
| `rm -rf ./client/.vite`                   | Удаление кэша Vite (помогает при ошибках импорта pmtiles)             |


##  Важные замечания (Troubleshooting)
*   **Ошибки при установки библиотек** Выключите впн))
*   **Производительность**: Первый запуск генерации тайлов может занять от 2 до 5 минут, в результате чего будет создан файл размером около 75 МБ.
*   **Ошибка "Wrong magic number"**: Если после перезаписи тайлов карта не загружается корректно, выполните следующие действия для очистки кэша и перезапуска фронтенда:

    ```
    rm -rf ./client/.vite
    docker compose restart frontend
    ```

    После этого обновите страницу в браузере, используя `Shift + Cmd + R` (для macOS) или `Ctrl + F5` (для Windows/Linux), чтобы принудительно сбросить кэш браузера.
